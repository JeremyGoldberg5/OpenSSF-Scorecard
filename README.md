## Overview
This repository was created to demonstrate OpenSSF security badges and basic secure repository practices. It includes a public GitHub project, a welcome page, and the required badge integrations for the lab assignment.

## Purpose
The main goal of this project is to:
- demonstrate the OpenSSF Scorecard badge
- demonstrate the OpenSSF Best Practices badge
- show a basic example of a repository with documentation and security-related project setup

## Badges Earned
This repository includes the following badges:
- **OpenSSF Scorecard**
- **OpenSSF Best Practices**

## Repository Contents
- `README.md` – project welcome page and badge display
- `LICENSE` – MIT license
- `.github/workflows/scorecard.yml` – GitHub Actions workflow for Scorecard analysis

## Technologies Used
- GitHub
- GitHub Actions
- OpenSSF Scorecard
- OpenSSF Best Practices BadgeApp

## How It Works
The OpenSSF Scorecard workflow runs through GitHub Actions and publishes repository security analysis results. The Best Practices badge is earned through the OpenSSF Best Practices program after meeting the required checklist items.

## Why This Project Matters
Software security and repository health are important in modern development. These badges help show that a project follows recognized open source security practices and is being evaluated with trusted standards.
